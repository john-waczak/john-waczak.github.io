Sagemath & Interacts

For a while now I have been using Mathematica as a computer algebra and vizualization package. It works very well but has an expensive licesnce if you don't have access to a free copy through school or some other institution. Last term, I was turned onto Sagemath by my differential geometry and general relativity teacher Dr. Tevian Dray. In the context of the class, we had been learning about the wedge product and exterior algebra when Tevian created a webpage on the class website that had cells similar to a Mathematica or Jupyter notebook that allowed us to apply ∧, *, and d to geometries that were excedingly challenging. The book he wrote on the subject is awesome by the way-- you should read it!

Anyway, the topic at hand is Sagemath and how we can embed it into webpages to make it easy for anyone to solve mathematics problems. Here I will follow the tutorial found in chapter six of this book . It is rather simple but showcases how we can embed cells to do cloud-based sage computations.

It turns out that embedding sage into a webpage is actually really easy! The way to do it is by creating Sage cells in your webpage that send any valid code you write off to a server to do the calculation. The details are explained here. The following example shows how we can make two simple functions-- the first is a sine wave and the second is its derivative (cosine)-- and then plot them.

The next step is to make our cells interactive. That means creating functions that use the @interact feature and control the type of inputs and outputs users can control (sliders, entering values, etc...).

The @interact keyword tells sage that this code should be run as an interactive cell. Then all we have to do is dump our code into a function and allow the parameters to vary over some range. The range of parameters is controlled, in this instance, by the selector function which allows the user to select which function to graph.
